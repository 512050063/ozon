# Ozon跨境电商助手 - 业务流程文档

---

## 📋 项目概述

### 项目名称
**Ozon跨境电商助手**

### 项目目标
为Ozon平台卖家提供一站式解决方案，包括选品分析、货源对接、商品管理和一键上架功能。

### 核心价值
- **自动化选品**：通过数据分析提高选品效率
- **货源对接**：快速匹配1688优质供应商
- **简化操作**：一键上架，减少重复工作
- **多语言支持**：自动俄语翻译，降低语言障碍

---

## 👥 用户角色

### 1. 管理员
- 管理系统用户
- 配置API权限
- 查看系统统计

### 2. 卖家（主要用户）
- 使用选品分析功能
- 管理商品库存
- 执行一键上架操作
- 配置个人API信息

---

## 🎯 核心业务流程

### 业务流程总图

```
用户登录 → 配置API → 选品分析 → 1688货源对接 → 本地仓库管理 → 一键上架
```

---

## 📖 详细业务流程

### 流程 1：用户认证流程

```
开始
  ↓
访问登录页面
  ↓
输入用户名和密码
  ↓
系统验证身份
  ↓
[验证成功？]
  ├─ 是 → 生成JWT Token → 跳转至首页
  └─ 否 → 显示错误信息 → 返回登录页面
```

**用例说明：**
- **前置条件**：用户账号已创建
- **后置条件**：用户成功登录，获取访问权限
- **异常处理**：密码错误5次后暂时锁定账号

---

### 流程 2：API配置流程

```
进入API配置页面
  ↓
查看当前配置状态
  ↓
选择配置类型
  ├─ Ozon API配置
  │   ↓
  │   输入Client ID和API Key
  │   ↓
  │   测试连接
  │   ↓
  │   保存配置
  │
  ├─ 1688 API配置
  │   ↓
  │   输入App Key和Secret
  │   ↓
  │   测试连接
  │   ↓
  │   保存配置
  │
  └─ 翻译API配置
      ↓
      选择翻译服务商
      ↓
      输入API密钥
      ↓
      测试翻译功能
      ↓
      保存配置
```

**用例说明：**
- **前置条件**：用户已登录
- **后置条件**：API配置保存成功，可以正常调用

---

### 流程 3：Ozon选品分析流程

```
进入选品分析页面
  ↓
选择分析维度
  ├─ 热门商品
  ├─ 新品上架
  ├─ 价格区间
  └─ 类目分析
  ↓
设置筛选条件
  ├─ 时间范围
  ├─ 价格范围
  ├─ 销量阈值
  └─ 评分要求
  ↓
执行分析请求
  ↓
显示分析结果
  ↓
选择感兴趣商品
  ↓
查看商品详情
  ↓
添加到候选列表
  ↓
[继续选品？]
  ├─ 是 → 返回分析页面
  └─ 否 → 完成选品
```

**数据字段说明：**
- 商品ID、标题、价格、销量、评分、评论数、上架时间
- 类目路径、卖家信息、历史价格趋势

---

### 流程 4：1688货源对接流程

```
进入1688货源页面
  ↓
方式选择
  ├─ 从候选列表导入商品
  └─ 手动搜索商品
  ↓
搜索1688货源
  ↓
显示匹配结果
  ↓
比较价格和质量
  ↓
选择优质货源
  ↓
查看供应商详情
  ↓
保存货源信息
  ↓
[继续找货？]
  ├─ 是 → 继续搜索
  └─ 否 → 完成货源对接
```

**数据同步说明：**
- 商品图片、价格、库存、规格参数
- 供应商联系方式、起订量、发货时间

---

### 流程 5：本地仓库管理流程

```
进入本地仓库页面
  ↓
查看商品列表
  ↓
操作选择
  ├─ 添加新商品 → 从选品列表或1688导入
  │   ↓
  │   填写商品信息
  │   ↓
  │   自动翻译标题/描述
  │   ↓
  │   保存商品
  │
  ├─ 编辑商品 → 修改价格、库存等信息
  │   ↓
  │   保存修改
  │
  ├─ 删除商品 → 确认删除 → 移除商品
  │
  └─ 查看详情 → 显示完整信息
  ↓
批量操作
  ├─ 批量编辑
  ├─ 批量删除
  └─ 批量导出
```

**商品状态说明：**
- **待上架**：已添加到仓库，未上架
- **已上架**：已同步到Ozon店铺
- **下架中**：在Ozon中已下架
- **已售罄**：库存为0

---

### 流程 6：一键上架流程

```
进入待上架商品列表
  ↓
选择要上架的商品
  ↓
检查商品信息完整性
  ↓
[信息完整？]
  ├─ 否 → 提示补充信息 → 返回编辑
  │
  └─ 是 → 设置上架参数
      ↓
      选择上架到Ozon的哪个店铺
      ↓
      设置价格策略
      ├─ 固定加价比例
      ├─ 自动计算汇率
      └─ 自定义价格
      ↓
      设置库存策略
      ├─ 与1688同步
      ├─ 固定库存值
      └─ 自定义库存
      ↓
      确认上架商品列表
      ↓
      执行一键上架
      ↓
      显示上架进度
      ↓
      查看上架结果
      ├─ 成功 → 更新商品状态为已上架
      └─ 失败 → 显示错误原因，支持重试
```

**上架策略说明：**
- 价格自动转换（人民币→卢布）
- 库存自动同步（定时更新）
- 图片自动处理（压缩、格式转换）

---

### 流程 8：自动翻译流程

```
触发翻译场景
  ├─ 添加新商品时
  ├─ 编辑商品时
  └─ 手动触发翻译
  ↓
识别需要翻译的文本
  ├─ 商品标题
  ├─ 商品描述
  ├─ 商品规格
  └─ 商品属性
  ↓
调用翻译API
  ↓
[翻译成功？]
  ├─ 是 → 填充翻译结果到对应字段
  └─ 否 → 保留原文，提示用户手动翻译
  ↓
用户可编辑翻译结果
  ↓
保存修改
```

**翻译规则说明：**
- 支持双语对照显示
- 支持常用短语自动替换（电商术语）
- 保留原始HTML格式


## 🛡️ 业务规则

### 规则 1：API调用限制
- Ozon API：遵循官方频率限制（通常为每分钟60次）
- 1688 API：按账户等级限制调用次数
- 翻译API：按套餐限制调用量

### 规则 2：商品状态变更
```
待上架 → 上架中 → 已上架 → 下架中 → 已下架
  ↓         ↓         ↓        ↓
编辑中   成功/失败  可下架   可重新上架
```

### 规则 3：价格计算
```
Ozon售价 = (1688价格 + 运费) × 汇率 × 利润率 × (1 + 平台费率)
```

### 规则 4：库存同步
- 默认每2小时同步一次库存
- 缺货商品自动下架提醒
- 支持手动立即同步

## 📝 业务场景示例

### 场景 1：新卖家首次使用流程
1. 注册账号并登录
2. 配置Ozon API凭证
3. 配置1688 API凭证
4. 配置翻译服务
5. 在Ozon进行选品分析
6. 在1688寻找货源
7. 添加商品到本地仓库
8. 选择商品一键上架

### 场景 2：日常选品上架流程
1. 登录系统
2. 查看首页统计，了解店铺运营情况
3. 进行Ozon选品分析
4. 对接1688货源
5. 商品信息完善和翻译
6. 批量上架商品
7. 检查上架结果

### 场景 3：库存管理流程
1. 查看库存统计
2. 发现库存不足商品
3. 在1688检查供应商库存
4. 更新本地仓库库存
5. 同步到Ozon店铺
